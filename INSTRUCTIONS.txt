The code in ./scripts generates GEMINI scores for the high-risk LUCAS cohort. Shell scripts were run using the Sun Grid Engine (SGE) computing system and 
and contain options that were passed to the 'qsub' command at the top of the script. For a detailed description of these options, please see
the SGE manual (https://gridscheduler.sourceforge.net/htmlman/manuals.html). Note that file paths in scripts are relative to the directory containing the script.
Software versions, including for R, python, and their packages are included in ./software_versions.

Note that reads in BAM files are assumed to be aligned to hg19. In addition, BAM files must be indexed (e.g. using samtools index)
such that each bam file has a corresponding index file in the bams directory (e.g. PGDX26283P4.sorted.bam and PGDX26283P4.sorted.bam.bai
both must be in the bams directory). For generating BAM files from FASTQ files, please see https://github.com/cancer-genomics/reproduce_lucas_wflow.
